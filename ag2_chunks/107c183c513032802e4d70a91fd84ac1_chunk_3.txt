 dialog with the data of the respective unit for subsequent editing.</p>
</li>
<li>
<p><strong>Create unit</strong>: Opens the dialog where you can enter the data for a new unit. By default, this unit is created below the respective unit.</p>
</li>
<li>
<p><strong>Export to Excel</strong>: Triggers the export of the respective unit and all subunits.</p>
</li>
<li>
<p><strong>Import structure</strong>: Allows the import of new substructures under the respective unit.</p>
</li>
<li>
<p><strong>Participants</strong>: Opens the <strong>Participants</strong> tab in the detailed view of the respective unit.</p>
</li>
<li>
<p><strong>Move up</strong> / <strong>down</strong>: Moves the respective unit and all its subunits up or down one position in its branch. Please note that this will merely modify the display while the structure remains unchanged.</p>
</li>
<li>
<p><strong>Adjust target figure</strong>: You can define a correction value that will be applied to the aggregated participant numbers of the respective unit.</p>
</li>
<li>
<p><strong>Delete</strong>: This option allows you to delete the respective unit while moving any existing participant data or subunits to another position.</p>
</li>
</ul>
<p class="wysiwyg-text-align-justify">These actions are located in a layer that opens when you click on the <strong>Actions</strong> icon.</p>
<p class="wysiwyg-text-align-justify"><strong>Actions for editing all or selected units</strong></p>
<p class="wysiwyg-text-align-justify">The following actions can be applied to selected or all units. They serve primarily for process control, e.g., for locking units against further editing or for sending e-mails to users<strong>.</strong></p>
<ul class="wysiwyg-text-align-justify">
<li>
<p>Lock units: With this action, you can protect units against further editing.</p>
</li>
<li>
<p>Lock units and subunits.</p>
</li>
<li>
<p>Release units: With this action, you can undo the locking of units.</p>
</li>
<li>
<p>Release units and subunits.</p>
</li>
<li>
<p>Mail users.</p>
</li>
<li>
<p>Mail users of the selected unit and its subunits.</p>
</li>
<li>
<p>Release units for filtering: Allows access to these units from the condition editor when defining filters and hiding conditions.</p>
</li>
<li>
<p>Revoke filter release for units: Deactivates access to these units from the condition editor.</p>
</li>
<li>
<p>Delete units: This action allows you to delete several units en bloc if these do not yet contain any participant data or subunits.</p>
</li>
</ul>
<p class="wysiwyg-text-align-justify">These actions can be found in the drop-down list below the organizational structure. Select the desired units by ticking the checkboxes in the “Actions” column or via the checkbox “Apply action to the whole organizational structure”. Then specify the desired action and confirm by clicking on <strong>Execute</strong>.</p>
<h3 id="detail-view" class="wysiwyg-text-align-justify"><strong>Detail view</strong></h3>
<p class="wysiwyg-text-align-justify">By clicking on a unit in the organizational structure you open its corresponding detail view. The detail view contains all the data stored for the unit, arranged on various tabs.</p>
<ul class="wysiwyg-text-align-justify">
<li>
<p><strong>Status information</strong>: This tab contains the following information and functions:</p>
<ul>
<li>
<p>Name, short form of name, and meta name of the unit. These data can be edited on the <strong>Edit unit</strong> tab.</p>
</li>
<li>
<p>Buttons for locking and releasing the unit or its subunits for editing.</p>
</li>
<li>
<p>Data that indicate the status of the survey for the unit and its subunits.</p>
</li>
</ul>
</li>
<li>
<p><strong>Edit unit</strong>: On this tab, you can subsequently edit basic properties of the unit such as name, meta name, or preset number of staff.</p>
</li>
<li>
<p><strong>Text elements</strong>: Unit-specific Org Processor data, such as names and other unit properties, can be inserted into the questionnaire using wildcards. On this tab, the desired data - and in multilingual surveys, their translation - are entered.</p>
</li>
<li>
<p><strong>Participants</strong>: On this tab, you create and manage the survey participants of the unit.</p>
</li>
<li>
<p><strong>User management</strong>: On this tab, you can view the responsible persons of the unit or define new responsible persons.</p>
</li>
<li>
<p><strong>Paper-pencil questionnaires</strong>: If the logistics functions of EFS Employees are activated, you can register your paper-pencil questionnaire requirements or view already existing requisition notes.</p>
</li>
<li>
<p><strong>Import structure</strong>: The import dialog on this tab allows the import of new substructures under the current unit.</p>
</li>
</ul>
<p class="wysiwyg-text-align-justify">The tabs of the detail view provide access to a wide range of data and functions. You will find detailed explanations in the respective chapters mentioned above.</p>
