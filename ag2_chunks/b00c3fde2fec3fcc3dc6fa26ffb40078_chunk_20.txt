-align-justify">Mails can be filtered by status by selecting the respective entry in the drop-down menu.</p>
<h3 id="viewing-project-specific-dispatch-processes-installation-wide" class="wysiwyg-text-align-justify"><strong>Viewing project-specific dispatch processes installation-wide</strong></h3>
<p class="wysiwyg-text-align-justify">The <strong>Project mail overview</strong> tab in the <strong>Options</strong> → <strong>Platform cockpit</strong> menu gives a central overview of all mail processes that can be initiated from the current installation.</p>
<ul class="wysiwyg-text-align-justify">
<li>
<p>You can display mail activity from any desired period in time.</p>
</li>
<li>
<p>Listed for each mail activity are the name of the project, precise dispatch date, project type, field time, e-mail subject, and number of mails submitted.</p>
</li>
<li>
<p>Excluded from the count are mail submission processes from deleted projects.</p>
</li>
<li>
<p>Mail activities are arranged by project.</p>
</li>
<li>
<p>Access to this tab is only possible for users with root or admin rights. These users may see mail activities for those projects for which their team holds read rights.</p>
</li>
</ul>
<p class="wysiwyg-text-align-justify">Read rights for <em>platform_cockpit</em> are required to view and open the <strong>Options</strong> → <strong>Platform cockpit</strong> menu.</p>
<h2 id="categories" class="wysiwyg-text-align-justify"><strong>Categories</strong></h2>
<p class="wysiwyg-text-align-justify">Under the menu item <strong>Categories,</strong> you can create both <strong>Default group</strong> and <strong>Rater group categories</strong>. Under the <strong>Default group </strong>you create global categories which you use to create thematic links between the projects. <strong>Rater group categories</strong> belong in the context of the <em><strong>Me </strong>Guide</em> scenario.</p>
<h3 id="default-group" class="wysiwyg-text-align-justify"><strong>Default group</strong></h3>
<p class="wysiwyg-text-align-justify">We have a new <strong>Categories</strong> are available within our <strong>Libraries</strong>. <strong>Libraries</strong> are accessible from the main top-right <strong>System</strong> menu. Categories allow you to create themes within your surveys/projects and can also be assigned to tickets within our <em>Action Board </em>portals module. Once in the <strong>Categories</strong> area, you can add a category using <strong>+ Category</strong>. The configuration area opens, where you assign an individual meta name, select a color, and deliver an optional translation. The meta name is mandatory and limited to 3 to 30 characters. Umlauts cannot be used. If you want to delete a category, you must make sure that all connections have been disconnected. By clicking on the edit icon, you can find out which connections are available for a category. Questions can be assigned to a category in the questionnaire editor and action tickets in the detail view. Before you can delete a category, however, you must undo each connection for which the trashcan icon is available in the <strong>library</strong> area.</p>
<p class="wysiwyg-text-align-justify"><img src="https://discoverxi-supportdesk.zendesk.com/hc/article_attachments/21015444583186" alt="Categories.jpg"></p>
<p class="wysiwyg-text-align-justify">You create default groups in each of the languages available in the drop-down list. You select a language by ISO code after clicking <strong>+ Category</strong>, enter the title in the desired language, and click <strong>Add</strong>.</p>
<p class="wysiwyg-text-align-justify">Path:</p>
<ul class="wysiwyg-text-align-justify">
<li>
<p><em>Libraries</em> → <em>Categories</em> → <em>Default group</em> (Tab)</p>

</ul>
<h3 id="rater-group-categories" class="wysiwyg-text-align-justify"><strong>Rater group categories</strong></h3>
<p class="wysiwyg-text-align-justify">You create the new Rater-Groups category used in the <em><strong>Me</strong></em> Guide scenario under <strong>Libraries</strong>. You will find there the categories: <strong>Self-Assessment</strong>, <strong>Manager</strong>, <strong>Peers,</strong> and <strong>Team</strong>. These cannot be configured either. However, you can create new ones by clicking on the <strong>+ category</strong> and assigning a unique meta name in the first step. Then you use the ISO code to specify the language of the title you want to assign. You save.</p>
