</strong></p>
<ul class="wysiwyg-text-align-justify">
<li>Rows that are no longer required or have become redundant can be deleted from the layout.</li>
<li>Delete a row by clicking the trashcan icon to the far right.</li>
</ul>

<p class="wysiwyg-text-align-justify"><span style="color: #333333;">Note that the columns together with the entire content will be deleted with a row. This action is final and cannot be undone.</span></p>
<h2 id="adding-a-new-page-module" class="sc-eNNmBn lmEGgd wysiwyg-text-align-justify"><strong>Adding a new page module</strong></h2>
<p class="wysiwyg-text-align-justify"><span style="color: #091e42;">Page modules can be added to columns. There are various page modules with specific purposes available. As an example, on a login page, you will need the </span><strong>Login/ Registration</strong><span style="color: #091e42;"> module.</span></p>
<ul class="wysiwyg-text-align-justify">
<li>To add a new page module, you click on <strong>+ New module</strong>.</li>
<li>A list of available modules opens.</li>
<li>Select a page module from the list.</li>
</ul>
<p class="wysiwyg-text-align-justify"><img src="https://discoverxi-supportdesk.zendesk.com/hc/article_attachments/21015069269522" alt="Modul list.png"></p>
<ul class="wysiwyg-text-align-justify">
<li>For a good overview, the page modules are divided into different categories by which you can filter and quickly find modules. </li>
<li>Via the search function, you can filter modules directly. </li>
</ul>
<p class="wysiwyg-text-align-justify">You can add various modules to a row, either into different columns or into the same. If more than one module is placed into one column, they will be displayed on the exterior portal view in the order they have been added. However, you can move the modules using the arrow icons for each module until all modules are located at the desired location.</p>
<p class="wysiwyg-text-align-justify">Some modules just need to be added, and no further settings are required, others can or must be further configured. Modules can be moved on page level (within or between rows) via drag &amp; drop or arrow buttons. The page modules are displayed collapsed initially. Detail settings for page modules will be visible when they are opened. Furthermore, most of the modules can be renamed.</p>
<h3 id="modules" class="wysiwyg-text-align-justify"><strong>Modules</strong></h3>
<p class="wysiwyg-text-align-justify"><span style="color: #091e42;">The standard modules are automatically available after the activation of </span><em>Portals.</em></p>
<p class="wysiwyg-text-align-justify"><strong>Discussionboard</strong></p>
<p class="wysiwyg-text-align-justify">The discussion board page module provides the panelist with a list of available discussions. The content must be defined in the detail settings. While discussions can be used in multiple modules and are not restricted to use in discussion board modules, discussion boards must be associated with at least one discussion created in the tab <strong>Content</strong>. A default discussion board is provided for the default Portals package.</p>
<p class="wysiwyg-text-align-justify"><strong>Discussion</strong></p>
<p class="wysiwyg-text-align-justify">This module displays single discussions when selected e. g. from the discussion board list, as described above. However, discussion modules are not restricted for use in discussion board modules only, as the source can be e. g. a search result, the discussion list, or a notification.</p>
<p class="wysiwyg-text-align-justify"><strong>Login/Registration</strong></p>
<p class="wysiwyg-text-align-justify">The login module can be displayed in three different ways:</p>
<ul class="wysiwyg-text-align-justify">
<li>as login form only</li>
<li>as registration from only</li>
<li>both as login and registration form</li>
</ul>
<p class="wysiwyg-text-align-justify">Details required for login must be defined in the access permission settings.</p>
<p class="wysiwyg-text-align-justify"><strong>Text and Multimedia</strong></p>
<p class="wysiwyg-text-align-justify">The text and multimedia module lets you present plain texts and pictures or videos to your panelist community. Once you start typing/ pasting content into the entry field, a box presenting a set of editing tools will appear, similar to tools known from text processing programs. You can drag and drop the box to position it anywhere within the window.</p>
<p class="wysiwyg-text-align-justify">You also have placeholders in the editor that allow you to display user-specific profile information and master data in the external view. You create variables in the sub-navigation field <strong>Settings</strong> under the tab <strong>Profile Settings</strong>. </p>
