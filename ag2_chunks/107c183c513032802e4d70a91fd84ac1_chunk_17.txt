<p class="wysiwyg-text-align-justify"><strong>Exporting return data</strong></p>
<p class="wysiwyg-text-align-justify">The return data can be exported. This may be useful, for example, if you want to evaluate or present the data using third-party applications.</p>
<ul class="wysiwyg-text-align-justify">
<li>
<p>There are two export options:</p>
<ul>
<li>
<p>Small export: This data record contains the aggregate values for all units (return overview: accumulated sum of the participants, accumulated return, accumulated return rate).</p>
</li>
<li>
<p>Expanded export: This data record lists all data in detail (return overview, return of the respective unit, and separate values for online and paper-pencil questionnaires).</p>
</li>
</ul>
</li>
<li>
<p>The export always comprises the current unit and all subunits to which you hold the relevant access rights.</p>
</li>
<li>
<p>The formatting of the Excel files allows them to be presented or relayed to third parties without further preparation.</p>
</li>
</ul>
<p class="wysiwyg-text-align-justify">To export the return data, locate the desired unit on the<strong> Return</strong> tab. Click on the <strong>Export</strong> icon. Select the desired export option and the character set, and confirm by clicking on <strong>Send</strong>.</p>
<p class="wysiwyg-text-align-justify">If you want users to be able to export return data, you need to assign the right f<em>ield_export_response_short</em> and/or <em>field_export_response_full</em> to the appropriate roles.</p>
<p class="wysiwyg-text-align-justify">When presenting or relaying survey data, please observe the data protection regulations applying to your employee survey.</p>
<h3 id="report-overview" class="wysiwyg-text-align-justify"><strong>Report Overview</strong></h3>
<p class="wysiwyg-text-align-justify">Upon completion of the survey, results reports are generated and made available for download to the responsible persons. The distribution process is usually managed directly in the <em>Org Processor</em> via the <strong>Report Overview</strong> tab. Your users can open the <strong>Report Overview</strong> tab only if it has been activated for their role in the current phase. To allow your users to open and edit not only their respective units but also the corresponding subunits, you need to activate the right structure_view_down for the appropriate roles.</p>
<ul class="wysiwyg-text-align-justify">
<li>
<p>The organizational structure, which opens automatically, will show you whether there are any reports for the individual units and if so, how many.</p>
</li>
<li>
<p>In the detail view, you will find the list of reports and the user management of the respective unit.</p>
</li>
<li>
<p>As a project manager, you can download all available reports.</p>
</li>
</ul>
<p class="wysiwyg-text-align-justify">Above the organizational structure, you will find the same features for modifying the display that you are already familiar with from the <strong>Organizational structure</strong> tab: Selection of the structure view, Search, <strong>Expand all</strong> / <strong>Collapse all</strong>, <strong>Activate auto-expansion</strong>, and <strong>View</strong>.</p>
<p class="wysiwyg-text-align-justify">The data in the report overview are updated every five minutes.</p>
