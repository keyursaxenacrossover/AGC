<h1 id="h_01HGKSMPRS9JVGC6E9EA3DB98K" class="wysiwyg-text-align-justify">Overview</h1>
<p class="wysiwyg-text-align-justify">This article provides comprehensive information about managing projects in EFS (Enterprise Feedback Suite). It covers various aspects of project management, including creating, organizing, archiving, and reactivating projects, as well as different project types and their features.</p>
<h1 id="h_01HGKSMPRSYYE7GMM7DZ0VCKXK" class="wysiwyg-text-align-justify">Information</h1>
<p class="wysiwyg-text-align-justify">The survey module focuses on a graphical interface that does not require any scripting skills.</p>
<p class="wysiwyg-text-align-justify">An intuitive graphical interface provides access to features that not only allow the creation of simple surveys but instead enable the user to create questionnaires that follow research methodology, that create a fluent participant experience and that trigger additional workflows to follow up or to create actions and alerts.</p>

<h2 id="the-project-list" class="wysiwyg-text-align-justify"><strong>The Project list</strong></h2>
<p class="wysiwyg-text-align-justify">By clicking the <strong>Projects</strong> button, you will get access to the <strong>Project list</strong>, the <strong>Field overview,</strong> and the <strong>Archive</strong>. The <strong>Project list</strong> will be opened by default.</p>
<p class="wysiwyg-text-align-justify"><img src="https://discoverxi-supportdesk.zendesk.com/hc/article_attachments/21017226439826" alt="Project list.jpg"></p>
<p class="wysiwyg-text-align-justify">The project list gives you an overview of all of the projects you have access to. You will find the following information on every project in the table:</p>
<ul class="wysiwyg-text-align-justify">
<li>
<p>Title</p>
</li>
<li>
<p>Globe icon: survey languages.</p>
</li>
<li>
<p>Author (except for forum discussions)</p>
</li>
<li>
<p>Project ID (except for forum discussions)</p>
</li>
<li>
<p>Type: Project type</p>
</li>
<li>
<p>Status: The current state of your survey project each with information on the date of the last update.</p>
</li>
<li>
<p>Created: The date on which the project was created.</p>
</li>
<li>
<p>Start: Date on which the survey started in the field.</p>
</li>
<li>
<p>Traffic-light icon: Signifies whether participants can access the survey. If the survey is active and in the field period, then a green arrow pointing upwards is visible. If the survey is not active and not in the field period, then a red arrow pointing downwards is visible.</p>
</li>
<li>
<p>Actions: Via the icons and checkboxes you can edit individual projects or all projects.</p>
</li>
</ul>
<p class="wysiwyg-text-align-justify">The table columns can be hidden or visible in order to achieve a better overview. Click on the <strong>Show column</strong> icons to make them visible, and click on the <strong>Hide column</strong> icons to hide them.</p>
<p class="wysiwyg-text-align-justify"><strong>Modifying Display Settings</strong></p>
<p class="wysiwyg-text-align-justify">You have various options for displaying the project list:</p>
<ul class="wysiwyg-text-align-justify">
<li>
<p>With <strong>Collapse, </strong>you can close all folders except the default folder so that only the hierarchy and not the content is visible. With <strong>Expand </strong>you can open all folders.</p>
</li>
<li>
<p>By clicking on the <strong>Show column</strong> icons you can display the columns “Author”, “Pages”, and “Created” with the creation date for the project and “Start” with the date of when the field phase started</p>
</li>
</ul>
<p class="wysiwyg-text-align-justify"><strong>Searching for Projects</strong></p>
<p class="wysiwyg-text-align-justify">The following filter and search options are available:</p>
<ul class="wysiwyg-text-align-justify">
<li>
<p>Type: Project type</p>
</li>
<li>
<p>Status: Project status</p>
</li>
<li>
<p>Search: A keyword search will always cover the project title, project directory, and author.</p>
</li>
<li>
<p>The content of the folder “Projects edited recently” is not searched, but always displayed in full.</p>
</li>
</ul>
