<p><strong>Overview:</strong></p>
<p>In Discover XI, administrators can control the maximum number of respondents who can be active in a survey simultaneously. This setting is crucial for managing system performance and ensuring that surveys run smoothly without overloading the system. If you need to adjust this limit, either to increase or decrease it, this guide will walk you through the steps.</p>
<p><strong>Problem:</strong></p>
<p>You want to change the maximum number of respondents who can be active at the same time in Discover XI, but youâ€™re not sure how to do it. This setting is system-wide, affecting all surveys and is applied uniformly.</p>
<p><strong>Solution:</strong></p>
<p>To change the limit for the maximum number of respondents active at the same time, follow these steps:</p>
<ol>
<li>
<p><strong>Access System Settings:</strong></p>
<ul>
<li>Navigate to the main dashboard of Discover XI.</li>
<li>Go to <code>System -&gt; Options -&gt; System Settings</code>.</li>
</ul>
</li>
<li>
<p><strong>Locate the Participation Limit Setting:</strong></p>
<ul>
<li>Within the System Settings, find the option labelled <code>Change Participation Limit</code>.</li>
</ul>
</li>
<li>
<p><strong>Adjust the Limit:</strong></p>
<ul>
<li>Click on <code>Change Participation Limit</code>.</li>
<li>Enter the new limit for the maximum number of active respondents. For instance, change from the default 100 to 10,000, depending on your requirement.</li>
<li>Save the changes.</li>
</ul>
</li>
<li>
<p><strong>Understand the System-Wide Impact:</strong></p>
<ul>
<li>Be aware that this setting is applied system-wide, meaning it affects all surveys within the platform.</li>
<li>The limit you set will undercut any existing installation-wide settings but cannot bypass the maximum number of participants per project.</li>
</ul>
</li>
<li>
<p><strong>Automatic Monitoring:</strong></p>
<ul>
<li>The system checks every 90 seconds whether the active respondents exceed the set limit. If exceeded, new participants will be rejected until the number falls below the limit.</li>
</ul>
</li>
</ol>
<p><strong>Summary:</strong></p>
<p>Changing the maximum number of respondents active at the same time in Duscover XI is straightforward but has a system-wide impact. Adjust this setting with caution to ensure it aligns with your overall survey needs.</p>

<ol>

<p><strong>What happens if the limit is exceeded?</strong></p>
<ul>
<li>New participants will be rejected until the number of active respondents drops below the limit.</li>
</ul>
</li>
<li>
<p><strong>Can I set different limits for different surveys?</strong></p>
