<h2 id="special-functions-for-sas-export" class="wysiwyg-text-align-justify"><strong>Special Functions for SAS Export</strong></h2>
<p class="wysiwyg-text-align-justify">The SAS export generates an SPSS Portable file, a text file with labels, and a macro that can be executed in SAS. The macro can be used to label the SPSS Portable file and convert it into an SAS data table.</p>
<p class="wysiwyg-text-align-justify">You only need SAS to edit the export files, SPSS is not required.</p>
<h3 id="notes-on-sas" class="wysiwyg-text-align-justify"><strong>Notes on SAS</strong></h3>
<p class="wysiwyg-text-align-justify">Since the SAS export is performed using an SPSS Portable file, it is subject to the technical default settings and restrictions of SPSS.</p>
<p class="wysiwyg-text-align-justify"><strong>Character set</strong></p>
<p class="wysiwyg-text-align-justify">From EFS 8.2, the SPSS export supports UTF-8 and all other character sets that can be selected in EFS, with the single exception of UTF-16.</p>
<p class="wysiwyg-text-align-justify"><strong>Missings</strong></p>
<p class="wysiwyg-text-align-justify">The default settings for missing values are interpreted as follows during SAS export:</p>
<ul class="wysiwyg-text-align-justify">
<li>
<p>“-77” is defined as a missing in both SPSS and SAS. Therefore, it is represented with a period (“.”) in SAS.</p>
</li>
<li>
<p>“-66”, “-99” and “0” are not defined as missings in SAS. Accordingly, they are interpreted and displayed as number values.</p>
</li>
</ul>
<h3 id="making-default-settings-for-the-further-processing-of-the-sas-files" class="wysiwyg-text-align-justify"><strong>Making Default Settings for the Further Processing of the SAS Files</strong></h3>
<p class="wysiwyg-text-align-justify">To apply the macro to the other files without any further configuration after the SAS export, you must make the following default settings in the <strong>Options</strong> → <strong>Personal preferences </strong>menu on the <strong>Common settings </strong>tab before exporting the data:</p>
<ul class="wysiwyg-text-align-justify">
<li>
<p>Standard directory for SAS exports on your computer: This directory path is copied into the SAS macro generated by the EFS Survey. Therefore, the compressed file must be located in this directory for you to apply the macro to it successfully. Default setting: c:\Documents and Settings\user name\Desktop\</p>
</li>
<li>
<p>SAS library in which the SAS macro generated by EFS Survey is to store the converted data: The SAS macro will store the data temporarily in this library. You can access and then save the data from there. Default setting: work. export, where “work” is the name of the library and “export” is the user-defined name of the file.</p>
</li>
</ul>
