<p class="wysiwyg-text-align-justify" data-renderer-start-pos="17200"><strong data-renderer-mark="true">Setting the threshold for the server load</strong></p>
<p class="wysiwyg-text-align-justify" data-renderer-start-pos="17243">The threshold for the load limit is specified by the platform administrator. (Platform administrators are all members of the “Administration” user team.) To do so, switch to the <strong data-renderer-mark="true">Options</strong> → <strong data-renderer-mark="true">System settings</strong> menu and make the setting in the “Load limit” area. At a value of “0”, the load limit will be switched off. If you wish to use the function, a value of “3” is recommended. In general, this value naturally depends on your server’s performance.</p>

<p class="wysiwyg-text-align-justify" data-renderer-start-pos="17825"><strong data-renderer-mark="true">Defining default survey messages for an installation</strong></p>
<p class="wysiwyg-text-align-justify" data-renderer-start-pos="17879">As an administrator you can determine what default set of messages is to be used across the installation when creating a new project. When administering an installation where English is predominantly used, this allows you to ensure, for example, that an English message set is always imported.</p>
<p class="wysiwyg-text-align-justify" data-renderer-start-pos="18174">To do so, navigate to the <strong data-renderer-mark="true">Options</strong> → <strong data-renderer-mark="true">System settings</strong> menu. In the “Default survey messages” field you can select one of the available message sets and then save it with <strong data-renderer-mark="true">Change</strong>.</p>
