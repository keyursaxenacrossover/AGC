 “write” rights. Afterwards, confirm by clicking <strong data-renderer-mark="true">Save</strong>.</p>
<h3 id="Configuring-the-Editor" class="wysiwyg-text-align-justify" data-renderer-start-pos="13091"><strong data-renderer-mark="true">Configuring the Editor</strong></h3>
<p class="wysiwyg-text-align-justify" data-renderer-start-pos="13115">You can decide yourself which languages you want to edit at the same time in the editor and how big the entry fields should be. If you translate only one or two languages, for example, it might make sense to display the text elements of all languages at once and edit them in parallel. If you have access to half a dozen or more languages, on the other hand, it might be more comfortable to display only a few languages at once.</p>
<ul class="ak-ul wysiwyg-text-align-justify" data-indent-level="1">
<li>
<p data-renderer-start-pos="13542">Click the <strong data-renderer-mark="true">View </strong>button.</p>
</li>
<li>
<p data-renderer-start-pos="13568">A layer opens, offering the following configuration options:</p>
<ul class="ak-ul" data-indent-level="2">
<li>
<p data-renderer-start-pos="13632">Number of rows in input fields (height): This field allows you to add rows to the open entry fields.</p>
</li>
<li>
<p data-renderer-start-pos="13735">Number of characters per row in input fields (width): This value allows you to change the width of the open entry fields.</p>
</li>
<li>
<p data-renderer-start-pos="13860">Languages: You can activate the languages that you want to edit or hide superfluous languages. The languages available are those which you’re authorized to access.</p>
</li>
</ul>
</li>
<li>
<p data-renderer-start-pos="14032">Afterward, confirm with <strong data-renderer-mark="true">Apply</strong>.</p>
</li>
</ul>
<h3 id="Search-and-Replace" class="wysiwyg-text-align-justify" data-renderer-start-pos="14067"><strong data-renderer-mark="true">Search and Replace</strong></h3>
<p class="wysiwyg-text-align-justify" data-renderer-start-pos="14087">The “Search and Replace” function allows you to search the text elements of a project for a certain keyword and to optionally change the text in one step.</p>
<div class="ak-editor-panel cc-tqp4ke wysiwyg-text-align-justify" data-panel-type="info">
<div class="ak-editor-panel__content">
<p data-renderer-start-pos="14244"><em>The search function covers all text elements which are accessible via the text element editor. Survey messages and wildcards cannot be searched.</em></p>
<p data-renderer-start-pos="14392"><em>&lt;img&gt; tags used in the HTML question type for example to insert an image, are not covered by the “Search and replace” function. The purpose of this restriction is to prevent inadvertent destruction of image linking. The <strong data-renderer-mark="true">Questionnaire editor </strong>→<strong data-renderer-mark="true"> Media check </strong>function is better suited to manage the linking of images.</em></p>
</div>
</div>
<p class="wysiwyg-text-align-justify" data-renderer-start-pos="14708"><strong data-renderer-mark="true">Defining search criteria</strong></p>
<p class="wysiwyg-text-align-justify" data-renderer-start-pos="14734">First, open the <strong data-renderer-mark="true">Search and replace </strong>menu and define your search criteria. Then click the <strong data-renderer-mark="true">Search </strong>button. The following search criteria can be used:</p>
<ul class="ak-ul wysiwyg-text-align-justify" data-indent-level="1">
<li>
<p data-renderer-start-pos="14883">Language version to be searched: You can search those languages which you are permitted to access.</p>
</li>
<li>
<p data-renderer-start-pos="14987">Search term</p>
</li>
<li>
<p data-renderer-start-pos="15004">Optionally, case sensitivity can be activated</p>
</li>
<li>
<p data-renderer-start-pos="15053">Optionally, you can search for regular expressions</p>
</li>
<li>
<p data-renderer-start-pos="15107">It is possible to search for specific text types. You can, for example, restrict your search to scale titles or long variable names.</p>
</li>
</ul>
<p class="wysiwyg-text-align-justify" data-renderer-start-pos="15243"><strong data-renderer-mark="true">Viewing the results</strong></p>
<p class="wysiwyg-text-align-justify" data-renderer-start-pos="15264">In the results table the results are listed, sorted by page, with the following information:</p>
<ul class="ak-ul wysiwyg-text-align-justify" data-indent-level="1">
<li>
<p data-renderer-start-pos="15360">Type: Text type</p>
</li>
<li>
<p data-renderer-start-pos="15381">Content: The text of the respective question, answer category or fill-in instruction. The search term is highlighted.</p>
</li>
