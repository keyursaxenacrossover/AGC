<p class="wysiwyg-text-align-justify"><strong>Systematically naming variables in dynamic answer blocks</strong></p>
<p class="wysiwyg-text-align-justify">The “Rename variables in dynamic answer blocks” option allows you to automatically change the names of variables, for example, in a list or a loop in such a way that the link to the original variable can be read from the name. If, for example, a dynamic list is incorporated into a question, all variables that were generated via the dynamic list will have the name v_sourcevariablea once renaming is complete. If the questionnaire contains a second question that draws on the same dynamic list, then the corresponding variables will be named v_sourcevariableb.</p>
<p class="wysiwyg-text-align-justify"><strong>Editing variable names externally</strong></p>
<p class="wysiwyg-text-align-justify">You can export external variable names, edit them outside EFS and then re-import them:</p>
<ol class="wysiwyg-text-align-justify" start="1">
<li>
<p>The “Export variable names for external editing” will create a CSV file containing the variable's information, which you can load onto your workstation. You can specify which character set is to be used.</p>
</li>
<li>
<p>When editing you enter the new external variable name in the “New varname” column.</p>
</li>
<li>
<p>The “Import externally edited variable names” option allows you to re-import the externally edited list in CSV format into EFS Survey. Here, too, the correct character set must be specified.</p>
</li>
</ol>
<p class="wysiwyg-text-align-justify"><strong>Undoing changes</strong></p>
<p class="wysiwyg-text-align-justify">Changes to external variable names can be undone by applying the “Restore original variable names” option.</p>
<h3 id="printing-and-exporting-codebook" class="wysiwyg-text-align-justify">Printing and Exporting Codebook</h3>
<p class="wysiwyg-text-align-justify">The codebook can be printed out or exported as a whole:</p>
<ul class="wysiwyg-text-align-justify">
<li>
<p>By clicking on the <strong>Print version </strong>icon in the codebook header you can call up your PC's print functions and print out the code book.</p>
</li>
<li>
<p>In the “Export” section of the codebook, you can download the codebook to your workstation. First set the desired character set, then choose the format – either CSV or Excel (xlsx) – and confirm with the <strong>Export </strong>button.</p>
</li>
</ul>

<h3 id="faq_1" class="wysiwyg-text-align-justify">What is the purpose of the Codebook?</h3>
<p class="wysiwyg-text-align-justify">The Codebook provides an overview of all variables and encodings used in a project, helping with control information, filters, triggers, and survey result analysis.</p>
<h3 id="faq_2" class="wysiwyg-text-align-justify">How can I access the Codebook?</h3>
<p class="wysiwyg-text-align-justify">Click on the <strong>Codebook</strong> menu item in the submenu for the respective function to open the Codebook in a new browser window.</p>
<h3 id="faq_3" class="wysiwyg-text-align-justify">Can I edit variable names in the Codebook?</h3>
<p class="wysiwyg-text-align-justify">Yes, you can edit variable names directly in the Codebook, use preset algorithms, or export and import variable names for external editing.</p>
