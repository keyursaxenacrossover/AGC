<h1 id="h_01HGKSMPRS9JVGC6E9EA3DB98K" class="wysiwyg-text-align-justify">Overview</h1>
<p class="wysiwyg-text-align-justify">This article provides a comprehensive guide on how to use the Task List module in Portals. It covers the configuration, setup, and usage of Task List for managing tasks within an organization.</p>
<h1 id="h_01HGKSMPRSYYE7GMM7DZ0VCKXK" class="wysiwyg-text-align-justify">Information</h1>
<p class="wysiwyg-text-align-justify">The Task List module in Portals allows users to create, view, and edit tasks without switching to the Org Processor's Action Planner. This guide explains how to set up and use Task List effectively.</p>
<ul class="wysiwyg-text-align-justify">
<li><a href="#configuring-task-list">Configuring Task List</a></li>
<li><a href="#working-with-task-list-in-the-exterior-view-of-portals">Working with Task List in the exterior view of Portals</a></li>
</ul>
<p class="wysiwyg-text-align-justify"><strong>Requirements for the use of </strong><em><strong>Task List</strong></em></p>
<p class="wysiwyg-text-align-justify">The following requirements must be met in order to use<em> Task List</em> for <em>Portals</em>:</p>
<ul class="wysiwyg-text-align-justify">
<li>
<p>Only ES type projects qualify for the use of a <em>Task List</em>.</p>
</li>
<li>
<p>Users creating, viewing, or editing tasks in the <em>Task List</em> must be allocated to the respective unit in the Org Processor first and must be assigned the correct role (e. g. owner).</p>
</li>
<li>
<p>Users working with <em>Task List</em> must have a panelist account based on the same e-mail address used in <em>Portals</em>.</p>
</li>
<li>
<p>The <em>Action Planner</em> must be activated first to use<em> the Task List</em> in <em>Portals</em>.</p>
</li>
<li>
<p>To use the page-module <em>Task List</em>, it must be added to a portal and configured first.</p>
</li>
</ul>
<p class="wysiwyg-text-align-justify"><strong>Adding </strong><em><strong>Task List</strong></em><strong> to a portal</strong></p>
<p class="wysiwyg-text-align-justify">To add a <em>Task List</em> to a portal, proceed as follows:</p>
<ul class="wysiwyg-text-align-justify">
<li>
<p>Open the portal to which you want to add the <em>Task List</em>.</p>
</li>
<li>
<p>Select a page on which you want to add the module.</p>
</li>
<li>
<p>Click on the button <strong>New row</strong> and select a layout from the default selection to define the number of columns of the new row.</p>
</li>
<li>
<p>Click on the button <strong>New module</strong>.</p>
</li>
</ul>
<p class="wysiwyg-text-align-justify">A list with all available page modules is opened.</p>
<ul class="wysiwyg-text-align-justify">
<li>
<p>Click on the page module<em> Task List</em>.</p>
</li>
</ul>
<p class="wysiwyg-text-align-justify">You have added the page module<em> Task List</em> to your portal.</p>
<h2 id="configuring-task-list" class="wysiwyg-text-align-justify">
<strong>Configuring </strong><em><strong>Task List</strong></em>
</h2>
<p class="wysiwyg-text-align-justify">To open the admin area of Task List, click on the arrowhead. The settings of the Task List are displayed. To configure the Task List, proceed as follows:</p>
<ul class="wysiwyg-text-align-justify">
<li>
<p>Select a<strong> Heading</strong> for the page module.</p>
</li>
<li>
<p>Select the <strong>Employee target project</strong> containing the tasks to be shown in the list display.</p>
</li>
<li>
<p>Using the checkbox “Users can create new tasks”, you can control if users can create new tasks or not. To do so, check or uncheck the box.</p>
</li>
</ul>
<p class="wysiwyg-text-align-justify">To create new tasks, responsible persons need to have at least the rights of the standard Owner role.</p>
<ul class="wysiwyg-text-align-justify">
<li>
<p>Define the <strong>number of tasks</strong> to be displayed on one page.</p>
</li>
<li>
<p>Define how many days before the expiration of a task a reminder will be sent out for a <strong>warning</strong>.</p>
</li>
</ul>
<p class="wysiwyg-text-align-justify">The expiration warning is set to one day by default.</p>
