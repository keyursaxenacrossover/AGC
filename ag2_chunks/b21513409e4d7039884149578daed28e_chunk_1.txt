Creating a Multi-Source Feedback Project</h2>
<p>If the new feedback project has not been created yet, start by creating a new project following the instructions in the “EFS Survey” documentation. Ensure you select the project type “Multi-source feedback”. Please note that the “multi-source feedback” project type (acronym: MSF) can only be used on EFS Leadership installations.</p>
<h3 id="h_01HGKSMPRV0YJ9R7XJXP9YDWO">Differences from Other EFS Project Types</h3>
<p data-renderer-start-pos="4439">A multi-source feedback differs from familiar EFS project types, as e.g. anonymous surveys and personalized surveys, in several basic aspects:</p>
<ul class="ak-ul" data-indent-level="1">
<li>
<p data-renderer-start-pos="4585">The menu <strong data-renderer-mark="true">Statistics</strong> is not available.</p>
</li>
<li>
<p data-renderer-start-pos="4626">The menu<strong data-renderer-mark="true"> Test and Validation → Project check</strong> is not available.</p>
</li>
<li>
<p data-renderer-start-pos="4692">The initial passwords of participants are visible (provided you have the necessary access rights). But after the first change, the passwords cannot be accessed or sent by e-mail anymore.</p>
</li>
<li>
<p data-renderer-start-pos="4882">Hostnames and IP addresses cannot be determined.</p>
</li>
</ul>
<h3 id="h_01HGKSMPRV5YJAR8YJXPAYDZP">Configuring Project Properties</h3>
<p>First, define the general framework data and settings for the new project.</p>
<ul>
<li>Configure general settings for all EFS project types in the Project Properties menu. This includes the survey URL, field time, survey messages, and languages. For detailed explanations, refer to the “EFS Survey” manual.</li>
<li>Configure settings specific to the multi-source feedback process in the Process configuration menu.</li>
</ul>
