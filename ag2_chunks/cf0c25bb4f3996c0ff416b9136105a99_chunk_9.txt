wysiwyg-text-align-justify" data-renderer-start-pos="16567">The various messages that are used in a survey, such as e.g. error messages, labels of HTML buttons, or of graphical form elements, are translated in the <strong data-renderer-mark="true">Survey messages </strong>menu.</p>
<div class="ak-editor-panel cc-tqp4ke wysiwyg-text-align-justify" data-panel-type="info">
<div class="ak-editor-panel__icon"> </div>
<div class="ak-editor-panel__content">
<p data-renderer-start-pos="16740">f graphical form elements are used in your project, you can change their labels in the <strong data-renderer-mark="true">Survey messages </strong>menu. The layout of graphical form elements is usually defined by the project manager, using the button generator in the admin area. You do not need access to it. In case manually-created graphics are used as graphical form elements in a project, for which you’re responsible, you can find the corresponding upload features in the <strong data-renderer-mark="true">Logos and form elements </strong>menu </p>
</div>
</div>
<p class="wysiwyg-text-align-justify" data-renderer-start-pos="17207">The dialog of this menu is divided as follows:</p>
<ul class="ak-ul wysiwyg-text-align-justify" data-indent-level="1">
<li>
<p data-renderer-start-pos="17257">The left-hand column, titled “Element”, explains in which situation the respective survey message will be used.</p>
</li>
<li>
<p data-renderer-start-pos="17372">The middle column contains the text elements of the default language, which is usually the reference language for the translation.</p>
</li>
<li>
<p data-renderer-start-pos="17506">The right-hand column shows the text elements in the target language. To change the target language, use the drop-down list “Which language do you want to edit” in the upper part of the dialog and confirm with <strong data-renderer-mark="true">Apply</strong>.</p>
</li>
</ul>
<div class="ak-editor-panel cc-tqp4ke wysiwyg-text-align-justify" data-panel-type="info">
<div class="ak-editor-panel__icon"> </div>
<div class="ak-editor-panel__content">
<p data-renderer-start-pos="17727">The character set in which the respective survey language will be encoded is defined in the field “Character set”. Usually, this is UTF-8. Change the setting of this field only after consultation with the project manager.</p>
</div>
</div>
<p class="wysiwyg-text-align-justify" data-renderer-start-pos="17951">To translate the survey messages, proceed as follows:</p>
<ul class="ak-ul wysiwyg-text-align-justify" data-indent-level="1">
<li>
<p data-renderer-start-pos="18008">If the desired target language is currently not displayed, select it in the drop-down list “Which language do you want to edit”.</p>
</li>
<li>
<p data-renderer-start-pos="18140">EFS contains a library in which complete sets of messages can be stored. If a complete set already exists in the library for the desired target language and the project manager has given you read access, you can upload these messages and use them as a base for your translation. To do so, select the appropriate set in the field “Copy elements from survey messages library”.</p>
</li>
<li>
<p data-renderer-start-pos="18516">Confirm with <strong data-renderer-mark="true">Apply</strong>. The desired target language is displayed. It contains the selected messages.</p>
</li>
<li>
<p data-renderer-start-pos="18616">Now, all preparations are finished, and you can start translating. When you’ve entered all texts, confirm with <strong data-renderer-mark="true">Save</strong>.</p>
</li>
</ul>
<h3 id="Translating-Wildcards" class="wysiwyg-text-align-justify" data-renderer-start-pos="18736"><strong data-renderer-mark="true">Translating Wildcards</strong></h3>
<p class="wysiwyg-text-align-justify" data-renderer-start-pos="18759">If wildcards are used to insert predefined texts into the questionnaire of your project, you can access and translate the source texts in the <strong data-renderer-mark="true">Wildcards </strong>menu. The editing dialog is structured quite similar to the editing pages of the text element editor.</p>
<h3 id="Uploading-Translated-Logos-and-Form-Elements" class="wysiwyg-text-align-justify" data-renderer-start-pos="19014"><strong data-renderer-mark="true">Uploading Translated Logos and Form Elements</strong></h3>
<p class="wysiwyg-text-align-justify" data-renderer-start-pos="19060">In multilingual projects, the layout created for the default language will usually be used for all other languages as well. However, you will sometimes wish to display customized layouts for the various languages (if, for example, the regional subsidiaries of a multinational company have different names and logos). The features that enable you to upload logos, background images and, if required, manually-generated buttons for the various languages can be found in the menu <strong data-renderer-mark="true">Logos and form elements</strong>. They allow to upload four logos, a background image, and various buttons for each language of the project.</p>
