<h2 id="general_conditions">General Conditions</h2>
<h3>For Setting Up a New Instance of EFS</h3>
<ul>
  <li>A new EFS instance must be set up when an opportunity has been created in Salesforce and Sales has informed 1st Level Support.</li>
  <li>Use the survey at <a href="http://ww2.efs-survey.com/uc/gp_interne_beantragungen/">http://ww2.efs-survey.com/uc/gp_interne_beantragungen/</a> to add a new instance.</li>
  <li>Optional additional features may include a new domain or SSL certificate.</li>
</ul>

<h3>For Removing an Existing Instance of EFS</h3>
<ul>
  <li>If a contract is terminated, Sales must inform 1st Level Support before the contract ends.</li>
</ul>

<p> </p>

<h2 id="required_information">Required Information</h2>
<ul>
  <li>To remove an instance or deactivate it, the following information is needed: date for deletion, instance, and organization.</li>
</ul>

<p> </p>

<h2 id="setup_process">Setting Up a New EFS Instance</h2>
<h3>Step 1: Initial Request</h3>
<p>1st Level Support is tasked by Sales to set up a new EFS instance.</p>

<h3>Step 2: Opportunity Check</h3>
<p>1st Level Support checks the corresponding opportunity in Salesforce for any additional appointed features.</p>

<h3>Step 3: Customer Clarification</h3>
<p>If necessary, 1st Level Support clarifies details with the customer.</p>

<h3>Step 4: Questionnaire Completion</h3>
<p>Sales, PS, or Support fills out the online questionnaire at <a href="http://ww2.efs-survey.com/uc/gp_interne_beantragungen/">http://ww2.efs-survey.com/uc/gp_interne_beantragungen/</a>.</p>

<h3>Step 5: Automated Notification</h3>
<p>An automatically generated email is sent to 1st Level Support.</p>

<h3>Step 6: Information Review</h3>
<p>1st Level Support checks for completeness and adds any additional information if needed.</p>

<h3>Step 7: Handover to Technical Support</h3>
<p>The information is handed over to Technical Support.</p>

<h3>Step 8: Instance Setup and User Account Creation</h3>
<p>After EFS Technical Support sets up the instance, 1st Level Support creates all requested user accounts and reports execution to the customer.</p>

<h3>Step 9: Completion Notification</h3>
<p>1st Level Support informs Sales that the task has been completed and can be invoiced.</p>

<p> </p>

<h2 id="removal_process">Removing an Existing EFS Instance</h2>
<p>For information on removing an existing EFS instance, please refer to the "Deletion of installations" guide.</p>

<h1 id="h_01HGKSMPRSER9ENY5QWWRZZ863">FAQ</h1>

<h3 id="faq_1">When should a new EFS instance be set up?</h3>
<p>A new EFS instance should be set up as soon as an opportunity has been created in Salesforce and Sales has informed 1st Level Support about it.</p>

<h3 id="faq_2">What information is needed to remove or deactivate an instance?</h3>
<p>To remove or deactivate an instance, you need the date for deletion, the instance name, and the organization details.</p>

<h3 id="faq_3">Who is responsible for informing 1st Level Support about contract termination?</h3>
<p>Sales is responsible for informing 1st Level Support before a contract ends if it is being terminated.</p>
