justify"><img src="https://discoverxi-supportdesk.zendesk.com/hc/article_attachments/21017191626642"><span style="font-size: 1.2em; font-weight: 600;">Configuring the consent form in the People module</span></p>
<p class="wysiwyg-text-align-justify" data-renderer-start-pos="17212">In the module People under <strong data-renderer-mark="true">Panel Configuration,</strong> you will find the menu item “Consent form”, where you can create the consent form for the purpose of registering a panelist. In order to be able to use this feature to capture the consent of future portal participants, you must configure the consent form in the default language, otherwise, the checkbox will not be displayed. One configuration applies to all portals. This feature is available both for the login dialog via the portal and for inviting other panelists via e-mail. Otherwise, the checkbox is not displayed.</p>
<ul class="ak-ul wysiwyg-text-align-justify" data-indent-level="1">
<li>
<p data-renderer-start-pos="17779">In the<em data-renderer-mark="true"> People</em> module, you have a language overview under the menu item <strong data-renderer-mark="true">Consent form configuration</strong> of the <strong data-renderer-mark="true">Panel configuration</strong>, so that you can immediately recognize in which languages the consent form has already been created.</p>
</li>
</ul>

<h2 id="Libraries:-Consent-form-templates" data-renderer-start-pos="18264">Libraries: Consent form templates</h2>
</div>
<p class="wysiwyg-text-align-justify" data-renderer-start-pos="18300">You do not have to configure the consent form for each project, but create consent forms under <strong data-renderer-mark="true">Libraries</strong>, which you will use when creating an consent form as required. Pre-filled forms depicting the most common use cases can also be found there, so that you have GDPR-compliant consent forms by means of minor but indispensable adjustments to your own project circumstances.</p>
<p class="wysiwyg-text-align-justify" data-renderer-start-pos="18676"> <strong>Path: <em data-renderer-mark="true">Navigation bar</em> → <em data-renderer-mark="true">System</em> → <em data-renderer-mark="true">Libraries</em> → <em data-renderer-mark="true">Consent form templates</em></strong></p>
<h3 id="h_01J3N0KQXZWMBSWZ7CX2FNB09V" class="wysiwyg-text-align-justify" data-renderer-start-pos="18676"><span style="font-size: 1.5em;">Templates overview</span></h3>
<p class="wysiwyg-text-align-justify" data-renderer-start-pos="18775">Under the menu item <strong data-renderer-mark="true">Consent forms templates</strong> you will find a search, the three buttons <strong data-renderer-mark="true">Create template</strong>, <strong data-renderer-mark="true">Import templates</strong> and <strong data-renderer-mark="true">Export all templates</strong> and an overview table of the created templates.</p>
<ul class="ak-ul wysiwyg-text-align-justify" data-indent-level="1">
<li>
<p data-renderer-start-pos="18971">You can edit, copy or delete existing form templates by clicking on the<strong data-renderer-mark="true"> pen</strong>, <strong data-renderer-mark="true">copy</strong> or <strong data-renderer-mark="true">delete</strong> icon in the right column. You change the permissions, i.e. who has access to the consent form template, by clicking on the <strong data-renderer-mark="true">permission</strong> icon.</p>
</li>
<li>
<p data-renderer-start-pos="19206">You have a language identifier that allows you to filter the consent form templates by language.</p>
</li>
</ul>
