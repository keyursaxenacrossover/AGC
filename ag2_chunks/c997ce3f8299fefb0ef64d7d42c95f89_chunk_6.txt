<h3 id="faq-1">What is the difference between RegEx and NoRegEx checks?</h3>
<p>RegEx checks return "true" and trigger an error message when the input matches the specified regular expression. NoRegEx checks do the opposite, returning "true" when the input does not match the specified regular expression.</p>
<h3 id="faq-2">How do I handle empty input fields in JavaScript plausibility checks?</h3>
<p>To prevent misevaluation of empty input fields in JavaScript plausibility checks, use a separate OnlyNumbers check instead of combining it with a Range check using AND conjunctions.</p>
<h3 id="faq-3">Can I use umlauts in e-mail address checks?</h3>
<p>No, umlauts are not allowed in any part of the e-mail address (username, domain name, or country ID) for e-mail address checks.</p>
