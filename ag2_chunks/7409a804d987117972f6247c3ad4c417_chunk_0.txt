<h1 id="h_01HGKSMPRS9JVGC6E9EA3DB98K">Overview</h1>
<p>This article provides information about the backup process, including data retention periods and the procedure for requesting and handling backup restorations.</p>

<p> </p>

<h1 id="h_01HGKSMPRSYYE7GMM7DZ0VCKXK">Information</h1>
<p>This guide covers the backup retention policy and the step-by-step process for handling backup restoration requests from customers.</p>

<ul>
  <li><a href="#backup-retention">Backup Retention</a></li>
  <li><a href="#restoration-process">Backup Restoration Process</a></li>
</ul>

<p> </p>

<h2 id="backup-retention">Backup Retention</h2>
<p>Backups are retained for a specific period to ensure data recovery capabilities:</p>
<ul>
  <li>Data backups can be precisely restored for each day of the <strong>last eight weeks</strong>.</li>
  <li>This information is sourced from the TECHNICAL &amp; ORGANIZATIONAL MEASURES FOR DATA PROTECTION document, version 1.55.</li>
</ul>

<p> </p>

<h2 id="restoration-process">Backup Restoration Process</h2>
<p>Follow these steps when handling a backup restoration request:</p>

<h3 id="step1">Step 1: Clarify Customer Requirements</h3>
<p>
  - Discuss with the customer which specific data needs to be restored.<br>
  - If the customer accidentally deleted data, <strong>always check the Admin-Log</strong> to verify the customer's claims.<br>
  - Document the results of the Admin-Log check in the ticket.
</p>

<h3 id="step2">Step 2: Create an Offer</h3>
<p>
  - Prepare an offer using the template located at: P:\ceur\GTM.CEUR\PS.CEUR\PS.SUPPORT.CEUR\04_Abrechnungen<br>
  - For Unipark: Use the Unipark ToDo file<br>
  - Use the file: Auftrag_DEUTSCH_2021-08.doc
</p>

<h3 id="step3">Step 3: Obtain Customer Approval</h3>
<p>
  - Have the customer sign the offer (For Unipark: Use Unipark ToDo)
</p>

<h3 id="step4">Step 4: Process Internal Order</h3>
<p>
  - Only after customer approval, fill out the Internal Order form (For Unipark: Use Unipark ToDo)<br>
  - Forward the case to technical support
</p>

<h3 id="step5">Step 5: Inform Customer</h3>
<p>
  - Once the mirror installation is available, notify the customer
</p>

<h3 id="step6">Step 6: Monitor Mirror Installation</h3>
<p>
  - The mirror installation is available for 14 days<br>
  - The case owner keeps the ticket open during this period and manages the deletion process
</p>

<h3 id="step7">Step 7: Delete Mirror Installation</h3>
<p>
  - After 14 days, forward the case to Technical Support for deletion of the mirror installation
</p>

<h3 id="step8">Step 8: Handle Invoicing</h3>
<p>
  - Don't forget to process the invoice. Refer to the "Invoicing in Netsuite (e.g. Mirror Installation, Training)" guide for details.
</p>

<p> </p>

<h1 id="h_01HGKSMPRSER9ENY5QWWRZZ863">FAQ</h1>

<h3 id="faq1">How long are the data backups retained?</h3>
<p>Backups for each day of the last eight weeks can be precisely restored.</p>

<h3 id="faq2">Where can I find the source document for backup retention information?</h3>
<p>The information is sourced from the TECHNICAL &amp; ORGANIZATIONAL MEASURES FOR DATA PROTECTION document, version 1.55. A link to the document in Dropbox is provided in the original guide.</p>

<h3 id="faq3">How long is the mirror installation available for the customer?</h3>
<p>The mirror installation is available for 14 days. After this period, it must be deleted.</p>
