">
<li>
<p data-renderer-start-pos="22511">Confirm by clicking on <strong data-renderer-mark="true">Save</strong>.</p>
</li>
</ul>
<p class="wysiwyg-text-align-justify" data-renderer-start-pos="22543"><strong data-renderer-mark="true">Translating text elements externally</strong></p>
<p class="wysiwyg-text-align-justify" data-renderer-start-pos="22581">You can open and edit the exported file in Excel or SDLX depending on which format you choose. </p>
<p class="wysiwyg-text-align-justify" data-renderer-start-pos="22677"><strong data-renderer-mark="true">Importing translated text elements back into EFS</strong></p>
<p class="wysiwyg-text-align-justify" data-renderer-start-pos="22727">After completing the translation, switch back to the EFS Translator interface and select the <strong data-renderer-mark="true">Import file </strong>tab in the menu <strong data-renderer-mark="true">Import / Export</strong>.</p>
<p class="wysiwyg-text-align-justify" data-renderer-start-pos="22862">Specify the target language and import conditions:</p>
<ul class="ak-ul wysiwyg-text-align-justify" data-indent-level="1">
<li>
<p data-renderer-start-pos="22916">In the drop-down list labeled “Into which language do you wish to import the new text elements?”, you can select the target language for the import.</p>
</li>
<li>
<p data-renderer-start-pos="23068">In the “Back up old elements in another language?” field, the “Overwrite old elements” function is selected by default. (If you do not want to lose text elements you have already entered, you can also back them up in another language. But when uploading translations, usually this does not make sense.)</p>
</li>
<li>
<p data-renderer-start-pos="23374">Only those elements containing a string will be transferred to the language table or the editor tables. Empty strings will be ignored.</p>
</li>
<li>
<p data-renderer-start-pos="23512">Use <strong data-renderer-mark="true">Browse </strong>to select the edited file.</p>
</li>
<li>
<p data-renderer-start-pos="23553">Click on <strong data-renderer-mark="true">Continue </strong>to import the changes.</p>
</li>
<li>
<p data-renderer-start-pos="23600">Changes will be documented in the overview “Import finished”.</p>
</li>
</ul>
<h3 id="Translating-Externally-with-Microsoft-Excel" class="wysiwyg-text-align-justify" data-renderer-start-pos="23665"><strong data-renderer-mark="true">Translating Externally with Microsoft Excel</strong></h3>
<p class="wysiwyg-text-align-justify" data-renderer-start-pos="23710"><strong data-renderer-mark="true">Editing the Excel file</strong></p>
<p class="wysiwyg-text-align-justify" data-renderer-start-pos="23734">The exported XLS file contains the text elements and survey messages of the questionnaire, an empty column for entering the desired translations, and various meta-information, ordered in a five or six-column table depending on the export settings.</p>
<p class="wysiwyg-text-align-justify" data-renderer-start-pos="23984">If you use external editing with Microsoft Excel in order to create translations into <em data-renderer-mark="true">several </em>languages, copy the exported file and use one file for each planned language version.</p>
