"><p><strong>Goal</strong></p></th>

<tr>
<th><p><strong>Q1</strong></p></th>
<td>
<p>The system should be highly available</p>
<p>(Reliability - High Availability)</p>
</td>
<td><p>The authentication should be used for every system in the porfolio of Tivian. It is a critical component every user and every system should use in future. A lack of availability means an outage for several to all parts of the system and should be avoided by any means.</p></td>
</tr>
<tr>
<th><p><strong>Q2</strong></p></th>
<td>
<p>The system should serve every connected system and user</p>
<p>(Performance Efficiency - High Capacity / Scalability)</p>
</td>
<td><p>The authentication should be used by every system and user. There will be a very large number or parallel users and possibly systems requesting authentication mechanisms. So Tivian Access has to be able to serve all of those.</p></td>
</tr>
<tr>
<th><p><strong>Q3</strong></p></th>
<td>
<p>The system should keep data secure and support authenticated access.</p>
<p>(Security - all)</p>
</td>
<td><p>The system contains user data and user credentials and takes care provides authentication mechanisms for other systems. Is is crucial for the system to only allow authorized access, prevent unauthorized modification of data and ensure accountability.</p></td>
</tr>
<tr>
<th><p><strong>Q4</strong></p></th>
<td>
<p>It should be easy to connect the system to other system for authentication mechanisms - <em>minor goal</em></p>
<p>(Compatibility - Interoperability)</p>
</td>
<td>
<p>The system should follow standards to connect other systems. Therefore, the ease to connect this system to others should be kept high enough to make the system useful.</p>
<p>This is still important but the first three goals are key to the success of the system.</p>
</td>
</tr>

<th><p><strong>Q5</strong></p></th>
<td>
<p>It should be easy for a user to use the authentication - <em>minor goal</em></p>
<p>(Usability - Operability)</p>
</td>
<td>
<p>A too complex authentication system will create confusion and misstrust for the users. It is important for arbitrary users to understand the system.</p>
<p>This is still important but the first three goals are key to the success of the system.</p>
