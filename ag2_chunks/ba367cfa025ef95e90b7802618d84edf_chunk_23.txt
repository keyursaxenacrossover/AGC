="color: #091e42;">offers settings concerning access rights. Several types of admissions for panelists can be set as well.</span></p>
<p class="wysiwyg-text-align-justify"><span style="color: #091e42;"><img src="https://discoverxi-supportdesk.zendesk.com/hc/article_attachments/21015039211538" alt="Access permissions.png"></span></p>
<p class="auto-cursor-target wysiwyg-text-align-justify">You start with the "Way of authentication for login" and choose between the three options via the drop-down list:</p>
<ul class="wysiwyg-text-align-justify">
<li>account</li>
<li>e-mail </li>
<li>e-mail or account</li>
</ul>
<p class="wysiwyg-text-align-justify">You continue with the "Panelist status". Several possible statuses are available for selection. You may also add custom panelist statuses. The options available are:</p>
<ul class="wysiwyg-text-align-justify">
<li>admitted temporarily</li>
<li>invited</li>
<li>Active</li>
<li>Tester</li>
<li>Moderator</li>
</ul>
<p class="wysiwyg-text-align-justify">The panelist statuses set here will be decisive for who exactly will be granted access to the portal. Generally, it will be every panelist in your database, however, only panelists matching the statuses activated in this setting will be able to access the portal. Panelist statuses are set in EFS People.</p>
<p class="wysiwyg-text-align-justify">At least you grant "Access permissions for panel groups". You define whether only group members or all registered panelists are granted access to your portal.</p>
<h3 id="assign-access-to-groups-only" class="sc-eNNmBn lmEGgd wysiwyg-text-align-justify"><strong>Assign Access to Groups only</strong></h3>
<p class="wysiwyg-text-align-justify">By default, all panelists will have access to your portal without further adjusting access rights.</p>
<p class="wysiwyg-text-align-justify">It is possible to further restrict access, based on panel groups.</p>
<ul class="wysiwyg-text-align-justify">
<li>Select <strong>Group members</strong> from the drop-down menu in the <strong>Access permission for </strong><strong>panel groups</strong> paragraph.</li>
<li>Click on <strong>+ Panel Group</strong>.</li>
</ul>
<p class="wysiwyg-text-align-justify">A drop-down menu appears.</p>
<ul class="wysiwyg-text-align-justify">
<li>Select a group and click the green checkmark icon.</li>
</ul>
<p class="wysiwyg-text-align-justify">Your group will be added. You can add as many groups as required. Delete groups from the list if necessary using the trashcan icon. Note that this action will only delete a group from the list, not the group itself. <span style="color: #333333;">Panel groups can be created in the EFS Panel and are a prerequisite for restricting access to group members. </span></p>
<p class="wysiwyg-text-align-justify">Variables can be selected to display specific information on a panelist‘s profile page of the exterior view. Variables can be selected and pre-defined labels can be renamed to meet your requirements. The label will later serve as a description for the defined information on a panelist‘s profile page. Define through a check box whether or not a panelist should be able to edit their personal information on their profile page.</p>
<h3 id="profile-settings" class="sc-eNNmBn lmEGgd wysiwyg-text-align-justify"><strong>Profile Settings</strong></h3>
<p class="wysiwyg-text-align-justify">If additional variables are required, you click on <strong>+ New Field</strong> to add another set of variables and label fields. The arrow icons will help you adjust the order of the variables as they will be displayed in a panelist‘s profile in the exterior view.</p>
<p class="wysiwyg-text-align-justify">You make the information about the panelists available to the <em>group</em> user so that they can be uniquely identified.</p>
<p class="wysiwyg-text-align-justify">Information</p>
<p class="wysiwyg-text-align-justify"><span style="color: #333333;">The panelist information must be released first in the EFS Panel (</span><strong>Panelist administration</strong><span style="color: #333333;"> → </span><strong>Profile Settings</strong><span style="color: #333333;">) before the information becomes available to use in Portals. The option </span><strong>Show profile to others</strong><span style="color: #333333;"> must be checked.</span></p>
