<h3 id="h_01J3G40MYYF28FF1DVWMQWJMB5">Setting Options in Detail</h3>
<h4 id="h_01J3G40MYYTCG9RR9M6YR9E65Q">Base Criteria</h4>
<p>You can limit the base of your panel statistics using the following criteria:</p>
<ul>
<li>
<strong>Panel Group Category</strong>: Defaults to the standard category if none is specified.</li>
<li>
<strong>Panel Group</strong>: Defaults to the standard panel group if none is specified.</li>
<li>
<strong>Language Version</strong>: Select the desired language version.</li>
<li>
<strong>Filter</strong>: Use grouping filters to refine your selection.</li>
<li>
<strong>Split Variables 1 and 2</strong>: Options include key system data (u_gender, pstatus, pinput, site_id, reg_code, pci) and all master data.</li>
<li>
<strong>Period of Entry into Panel</strong>: Define the time frame for panel entry.</li>
<li>
<strong>Panel Status</strong>: You must select at least one panel status, defaulting to “active” if none is checked.</li>
</ul>
<h4 id="h_01J3G40MYYTCVBC9PWYQ3WSA2E">Display Options</h4>
<p>Customize how results are displayed:</p>
<ul>
<li>
<strong>Multiple Variables Display</strong>: Choose whether results are displayed in a single or separate table blocks. For a single table block, check "Display results for all variables in the same table block".</li>
<li>
<strong>Display Modes for Multiple Split Variables</strong>:
<ul>
<li>
<strong>Nested</strong>: Shows all possible combinations of split variable characteristics for each characteristic of the analyzed variable.</li>
<li>
<strong>Side-by-Side</strong>: Separately counts frequencies and sums of characteristics for split variables, showing percentages.</li>
<li>
<strong>Cartesian Product</strong>: Lists frequency and percentage for each combination of characteristics from the analyzed variable and split variable(s).</li>
</ul>
</li>
</ul>
<h4 id="h_01J3G40MYYN4R5JN3P9FXT0A5T">Performance Data</h4>
<p>There is a variety of important performance data at your disposal: u_gender, pstatus, pinput, md_status, md_num_updates, md_num_invites, reg_code, site_id, and pci.</p>
<p>Additionally, you can also use all tracking variables.</p>
<h4 id="h_01J3G40MYYKPPCV7545CX2CTD4">Master Data</h4>
<p>All master data with answer categories are available, and you can limit the list to specific master data categories for clarity. Text variables can be made available for specific analysis scenarios upon request to support.</p>
