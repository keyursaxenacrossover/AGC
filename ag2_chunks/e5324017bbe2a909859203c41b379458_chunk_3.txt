<h3 id="Settings-on-EFS-Admin-and-Portals" data-renderer-start-pos="1058"><strong data-renderer-mark="true">Settings on EFS Admin and Portals</strong></h3>
<h4 id="What-settings-can-be-configure-in-EFS-?" data-renderer-start-pos="1093"><strong data-renderer-mark="true">What settings can be configure in EFS ?</strong></h4>
<p data-renderer-start-pos="1134">The module <strong data-renderer-mark="true">IdentityAccessManager</strong> has to be configured and activated in order to delegate authentication into EFS to Okta. This is the only places for configuration.<br>The module is added automatically with the 19.5 release to every installation per default.</p>
<p data-renderer-start-pos="1391">All settings will be configured by 2nd level support.</p>
<h4 id="How-can-I-configure-the-module-IdentityAccessManager" data-renderer-start-pos="1446"><strong data-renderer-mark="true">How can I configure the module IdentityAccessManager</strong></h4>
<p data-renderer-start-pos="1500">The module configuration can be found at <code class="code cc-1o5d2cw" data-renderer-mark="true">System &gt; Options &gt; Platform cockpit &gt; System configuration</code> in the EFS Admin UI.<br>All configuration options can be viewed and changed in the grouped collapsible <strong data-renderer-mark="true">IdentityAccessManager</strong>.</p>
