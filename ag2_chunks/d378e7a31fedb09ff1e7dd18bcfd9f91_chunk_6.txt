<h3 id="access-restrictions" class="wysiwyg-text-align-justify"><strong>Access Restrictions</strong></h3>
<p class="wysiwyg-text-align-justify">On the <strong>Access Restrictions</strong> tab, you can define role- and phase-specific restrictions for the individual pages of the download platform.</p>
<p class="wysiwyg-text-align-justify"><img src="https://discoverxi-supportdesk.zendesk.com/hc/article_attachments/21013838186770" alt="Access restrictions.jpg"></p>
<h3 id="overview-of-existing-access-restrictions" class="wysiwyg-text-align-justify"><strong>Overview of existing access restrictions</strong></h3>
<ul class="wysiwyg-text-align-justify">
<li>
<p>If a role-dependent access restriction has been defined, you can find the roles with access rights in the “Restrictions of Access” column.</p>
</li>
<li>
<p>The first traffic light icon in the “Properties“ column shows if the page is active.</p>
<ul>
<li>
<p>Green = active</p>
</li>
<li>
<p>Red = not active</p>
</li>
</ul>
</li>
<li>
<p>The second traffic light icon shows if the participant must be logged in.</p>
<ul>
<li>
<p>Green = needs to be logged in</p>
</li>
<li>
<p>Red = it is not necessary to log in</p>
</li>
</ul>
</li>
</ul>
<h3 id="changing-access-restrictions" class="wysiwyg-text-align-justify"><strong>Changing access restrictions</strong></h3>
<ul class="wysiwyg-text-align-justify">
<li>
<p>Click the <strong>Edit</strong> icon.</p>
</li>
<li>
<p>Use the field “Only for logged-in persons”, you define if a participant must be logged in to see the page.</p>
</li>
<li>
<p>In the field “Is the page active?”, you define if the page should be available at all. If a page is not active, a participant trying to open it will be routed to the template “Error message ‚page not found’”.</p>
</li>
<li>
<p>If you want to define a role-specific restriction, click the <strong>Add</strong> button.</p>
</li>
<li>
<p>Select the role that should have access.</p>
</li>
<li>
<p>Decide in which status of the the focus person the selected role should have access.</p>
</li>
<li>
<p>Confirm by clicking<strong> Save</strong>.</p>
</li>
</ul>
<h3 id="creating-access-restrictions-for-additional-pages" class="wysiwyg-text-align-justify"><strong>Creating access restrictions for additional pages</strong></h3>
<ul class="wysiwyg-text-align-justify">
<li>
<p>Click the <strong>Create page</strong> button.</p>
</li>
<li>
<p>Enter the name of the page.</p>
</li>
<li>
<p>Decide if the page should be available only for logged-in persons.</p>
</li>
<li>
<p>Decide if the page should be active.</p>
</li>
<li>
<p>Confirm by clicking <strong>Save</strong>.</p>
</li>
</ul>
<p class="wysiwyg-text-align-justify">You need to write permission for <em>employee_access_restrictions</em> to access the <strong>Access Restrictions</strong> tab in an employee survey.</p>
