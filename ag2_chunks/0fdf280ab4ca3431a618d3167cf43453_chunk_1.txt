<h2 id="pretest-comments" class="wysiwyg-text-align-justify"><strong>Pretest comments</strong></h2>
<p class="wysiwyg-text-align-justify">In the documentation area of the respective project, under <strong>Projects</strong> → {Selected project} → <strong>Documentation → Pretest comments</strong>, you can view the testers’ comments.</p>
<p class="wysiwyg-text-align-justify"><img src="https://discoverxi-supportdesk.zendesk.com/hc/article_attachments/21017226429586" alt="Pretest comments.jpg"></p>
<p class="wysiwyg-text-align-justify">The “Extended search”, which you can open by clicking on the link, provides various options for searching the comment list and restricting the results displayed. For example, you can search for a keyword from the comment or the page title. After the search is completed, you can tick the checkbox in the “Clear search form” field to return to the full comment list.</p>
<p class="wysiwyg-text-align-justify">The pretest comments will be shown in the table. The following additional information is available for each comment:</p>
<ul class="wysiwyg-text-align-justify">
<li>
<p>ID (of the comment)</p>
</li>
<li>
<p>Inserted on: creation date</p>
</li>
<li>
<p>User ID</p>
</li>
<li>
<p>Consecutive number: The testers are assigned consecutive numbers.</p>
</li>
<li>
<p>Contact information: In the case of personalized projects, the names and e-mail addresses of the testers will be taken from the participant's administration. For an anonymous project, contact information will be displayed only when capturing has been enabled and the testers have entered data.</p>
</li>
<li>
<p>Page ID: The ID of the questionnaire page.</p>
</li>
<li>
<p>Page title: By clicking on the page title, you can open the commented pages directly in the questionnaire editor.</p>
</li>
<li>
<p>Comment</p>
</li>
<li>
<p>Language: The language of the questionnaire in multilingual projects that was tested.</p>
</li>
<li>
<p>Browser: The browsers used by testers will be logged. This makes it easy for you to check, for example, whether layout tests have been performed with all major browsers or whether an error message is browser-specific.</p>
</li>
<li>
<p>Status: You can assign processing statuses to pretest comments. This facilitates coordination, particularly if you are preparing the project together with other users. You can select the statuses “not decided”, “to do”, “done” and “not relevant”.</p>
</li>
</ul>
<p class="wysiwyg-text-align-justify">Furthermore, the “Actions” column provides the following icons:</p>
<ul class="wysiwyg-text-align-justify">
<li>
<p><strong>Page preview</strong>: Shows the commented questionnaire page in the preview.</p>
</li>
<li>
<p><strong>Edit page</strong>: Opens the commented questionnaire page in the questionnaire editor.</p>
</li>
<li>
<p><strong>Edit language elements</strong>: Opens the commented questionnaire page in the language element editor.</p>
</li>
<li>
<p><strong>Delete comment</strong>.</p>
</li>
</ul>
