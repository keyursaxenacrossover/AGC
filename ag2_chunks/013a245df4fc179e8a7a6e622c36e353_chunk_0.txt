<h2 id="header-1"><strong>TIVIAN Access</strong></h2>
<p>To increase the security of your applications - after the implementation of the <em>Privacy Assistant</em> (GDPR) - the new <em>TIVIAN Access</em> module has been added. By the use of this module, that can be integrated into your existing SSO infrastructure, bad password habits will be mitigated. From now on, users must enter the username and password to access all the applications.</p>
<h3 id="header-3"><strong>Single Sign On</strong></h3>
<p>Using the<em> Single Sign-On</em> function, you only need to enter a user name and a password to access both the<em> Enterprise Feedback Suite</em> and <em>Portals</em>. It is no longer necessary to store different user names and passwords to access to this applications. The participant administration has been modified. First name, last name and e-mail address are mandatory fields after activation of the <em>TIVIAN Access</em> module.</p>
<h3 id="header-5"><strong>Multi Factor Authentication</strong></h3>
<p>Optionally, the multi-factor authentication can be added to the <em>Single Sign On</em> function. This means that in addition to entering the username and the password, you have to enter a six-digit code, which is mandatory. This code will be sent to you via authenticator apps such as the <em>Google Authenticator</em>.</p>
<p> </p>
<p>By default <em>TIVIAN Access</em> is disabled. So the login works as you are used to. If you want to enable <em>TIVIAN Access</em>, please contact our support team.</p>
