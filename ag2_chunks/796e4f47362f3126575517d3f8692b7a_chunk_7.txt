: Pages</strong></h2>
<p class="wysiwyg-text-align-justify">You execute the button <strong>+ New Page</strong> on the field <strong>Pages</strong> to create a new page in a Report Definition. A message will inform you about the success of the action. Your intention to delete a page requires a modal confirmation. This way, you won't accidentally delete a page. After successfully adding a page, you use the selection box to select a master slide from either all libraries created or a special one. In this way, you can compile your report individually. In addition to that:</p>
<ul class="wysiwyg-text-align-justify">
<li>You sort the page by drag &amp; drop.</li>
<li>You create a new page for each additional slide.</li>
<li>You find more about it under the <strong>master slide</strong>:
<ul>
<li>File format and file name</li>
<li>Library</li>
<li>Mapping Status</li>
</ul>
</li>
<li>You have two icons that allow you to download files and copy the mapping from another page.</li>
<li>You add conditions for each page of your report.</li>
<li>In the lower part of the page, dedicate yourself to the cells and tables to create the mappings.</li>
</ul>

<h3 id="create-mapping" class="auto-cursor-target wysiwyg-text-align-justify"><strong>Create mapping</strong></h3>
<p class="wysiwyg-text-align-justify">You click on <strong>Create mapping</strong> in one of the shape cells you want to start with and the inspector opens. There are two taps:<strong> Mapping</strong> and <strong>Filter</strong>. You do not set filters until you have created a mapping. The mapping starts by selecting a starting point for the picker path from up to four elements:</p>
<ul class="wysiwyg-text-align-justify">
<li> Unit</li>
<li>Dimension</li>
<li>Collections
<ul>
<li>Collections must first be created on the sub-navigation field <strong>Data </strong>under the tab <strong>Collections</strong> so that they can be used for the mapping.</li>
</ul>
</li>
<li>Text</li>
</ul>
<p class="wysiwyg-text-align-justify"><span style="color: #091e42;">You use these elements depending on your objectives. The paths on which you move from which starting element so that you fill the shape cells appropriately depends on the specifications of the report to be filled out.</span></p>

<p class="auto-cursor-target wysiwyg-text-align-justify"><strong>Layout of the Inspector I: Mapping</strong></p>
<p class="auto-cursor-target wysiwyg-text-align-justify"><span style="color: #091e42;">The inspector is one of the important controls in the </span><em>Report Manager</em><span style="color: #091e42;">. You use it when creating DSEs and collections. You also use it for the mapping. Generally speaking, it can be said that the inspector is the control panel moving from the right into the display.</span></p>
<p class="wysiwyg-text-align-justify">On the sub-navigation field <strong>Pages</strong>, the inspector moves in as soon as you click on <strong>Create mapping</strong>, and is used for the cartography. You define the paths that define how the shape cell is to be filled.</p>
<ul class="wysiwyg-text-align-justify">
<li>On this sub-navigation field, the inspector has two tabs:<strong> Mapping</strong> and <strong>Filter</strong>.</li>
<li>Under the two tabs, you will find information about which shape area you can map.</li>
<li>Each path is validated by setting a green check mark.</li>
<li>A red warning triangle indicates any pending validations.</li>
<li>You have a search function under the tap <strong>Mapping</strong> that allows you to reduce the more extensive selection options of the picker elements to a clear number. Use the arrow keys to navigate either by scrolling through the list of results block by block of ten or by jumping directly to the end of the beginning.</li>
</ul>
