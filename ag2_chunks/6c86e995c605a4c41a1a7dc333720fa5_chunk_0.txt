<h1 id="header-1">Portals</h1>
<h2 id="header-3">New page module <em>Reports</em>
</h2>
<p>The new <em>Reports </em>page module enables the display of reports which are released or requested for a certain user. The source of these reports are corresponding ES projects on the installation. The user will be informed about the availability of new reports in the notification center of Portals.</p>
<p><img src="https://discoverxi-supportdesk.zendesk.com/hc/article_attachments/20998615774354" alt="e0330d25-bc45-4ce3-b381-853c0c6d003a"></p>
<h3 id="header-5">Import / export of portal languages</h3>
<p>User-defined translations for standard phrases of a portal can now be exported and imported. This way, you’ll be able to export your data, send them to a translation agency for example and import the translated data into the system later.</p>
<h3 id="header-6">Layout limitation for page modules</h3>
<p>When configuring pages in the admin area of a portal, certain modules will only be available for specifically adapted layouts.</p>
<p>This way, misconfigurations can be avoided. Furthermore, modules are always being displayed in the optimal view.</p>
<h3 id="header-7">New indication of changes to pages in Portals admin area</h3>
<p>To prevent admins from forgetting to publish changes after editing pages, we built an eye-catching header, which suggests the publication of a changed page.</p>
<p><img src="https://discoverxi-supportdesk.zendesk.com/hc/article_attachments/20998586233746" alt="c1c1ec69-231d-41cb-ad01-040f1c8201e7"></p>
<h3 id="header-8">New user icon for <em>moderator</em>
</h3>
<p>A crown at the user icon now marks portal users as <em>moderator</em>. A corresponding quick info additionally indicates a user as <em>moderator </em>by mouseover.</p>
<h3 id="header-9">Rearrangement of tab <em>Contents</em>
</h3>
<p>The tabulation of the discussion board of the tab <em>Contents </em>within the Portals admin area has been optimized and adapted to the design of the other tabs. Besides, the content of the tab <em>Classifications </em>can be found on the tab <em>Contents </em>from now on.</p>

<h2 id="header-11">Action Planner enhancements</h2>
<h3 id="header-13">MyTasks Action Planner module for Portals</h3>
<p>In Portals you can add the <em>MyTasks </em>page module. With this new page module, you can create, view and edit actions. Heading, the target ES project and the number of actions can be defined in the Portals admin area. You can restrict all users from creating actions on their own by unchecking the provided option.</p>
