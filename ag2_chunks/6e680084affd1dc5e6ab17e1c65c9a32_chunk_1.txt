<h2 id="header-9">Portals</h2>
<h3 id="header-11"><em>Feedback Assistant</em></h3>
<p>The module <em>Feedback Assistant </em>(formerly named <em>On-Demand Guided) </em>for creating template-based ad hoc surveys was completely revised and is now available in an extended scope.</p>
<h3 id="header-12">Template configuration in personalized project</h3>
<p>The new template configuration for <em>Feedback Assistant </em>can be found in the respective project under “Project properties” and is divided into different areas.</p>
<p>The access to each template configuration is controlled via the rights and role system of EFS. EFS administrators with respective rights can extensively configurate each personalized project as templates for use and choice within the module <em>Feedback Assistant</em>.</p>
<p><strong>Basic information</strong></p>
<p>Here, administrators define a title as well as a description and a background image to make your template in the module <em>Feedback Assistant </em>visible.</p>
<p><img src="https://discoverxi-supportdesk.zendesk.com/hc/article_attachments/20998560751506" alt="34729a45-116e-4054-a724-233a0f105765"></p>
<p><strong>Synchronizing panelist data</strong></p>
<p>Via the new functionality “Synchronize panelist data”, you have the possibility to keep the data of panelists as participants of <em>Feedback Assistant </em>surveys up- to-date at all times.</p>
<p><strong>Selecting and creating context variables</strong></p>
<p>Select or create variables you want to offer to the survey manager within the module <em>Feedback Assistant </em>for free definition. Accordingly selected and filled variables will be available for evaluation in the dashboard afterwards.</p>
<p><strong>Selecting panelist variables</strong></p>
<p>If panelists take part in your survey, here you can select the panelist specific variables you want to be available for evaluation in the dashboard of the respective <em>Feedback Assistant </em>survey for further use.</p>
<p><strong>Selecting manager variables</strong></p>
<p>Master data variables which are solely allocated to the respective survey manager initiating the <em>Feedback Assistant </em>survey via the respective module in Portals can be selected in this section and appended to the template for later evaluation in the dashboard.</p>
<p><strong>Allocating the template to certain portals</strong></p>
<p>By selecting certain portals, you have the possibility to control in which portals your template will be available within the module <em>Feedback Assistant</em>.</p>
<p><strong>Defining mail templates for invitations and reminders</strong></p>
<p>You have the possibility to allocate mail templates of type “Projects” to the template to be configured, for invitation mails as well as reminder mails for participants of the <em>Feedback Assistant </em>survey.</p>
<h3 id="header-13">Portals module <em>Feedback Assistant </em>Overview – filtering, searching, sorting</h3>
<p>On the overview page of the module <em>Feedback Assistant</em>, you can now search for created <em>Feedback Assistant </em>surveys, filter the status of the surveys and sort the view by creation date.</p>
