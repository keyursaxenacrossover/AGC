<h3 id="granting-access-rights-to-macros" class="wysiwyg-text-align-justify"><strong>Granting access rights to macros</strong></h3>
<p class="wysiwyg-text-align-justify">To be able to define macros and to determine who will be allowed to execute them, a user must belong to a team that has write rights for<em> create_macro</em>. You can use the <strong>Rights</strong> icon in the overview to assign write or read rights for specific macros to individual teams.</p>
<h3 id="checking-the-application-of-macros" class="wysiwyg-text-align-justify"><strong>Checking the application of macros</strong></h3>
<p class="wysiwyg-text-align-justify">The <strong>Macro log</strong> menu logs all macro executions.</p>
<h2 id="question-types" class="wysiwyg-text-align-justify"><strong>Question types</strong></h2>
<p class="wysiwyg-text-align-justify">If you want to create user-defined question types not only temporarily, but add them to the library, you can do this in the menu <strong>Libraries</strong> → <strong>Question types</strong>. The newly defined question types are available across all projects in the Questionnaire Editor. You can add or import a new question type from another existing <em>EFS</em> installation. Click the <strong>New</strong> button or the drop-down menu →<strong> Import</strong>.</p>
<ul class="wysiwyg-text-align-justify">
<li>
<p>Enter the title of your new question type and select the appropriate category. Assign your newly created question type to one of the following categories of question types: <strong>Single response list</strong>, <strong>Multiple response list</strong>, <strong>Standard matrix</strong>,<strong> Text field,</strong> or <strong>911 special</strong>. Also in the Questionnaire Editor, you can find the newly created question type in the selected category.</p>
</li>
<li>
<p>The newly created question type will be listed in the overview. To edit it, click on its title. This opens an HTML editor that allows you to create a new question type. Here you can customize the template, the stylesheet, the Javascript, the exact settings, and the resources.</p>
</li>
</ul>
