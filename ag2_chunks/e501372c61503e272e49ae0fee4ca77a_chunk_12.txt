>– To open a forum discussion from the project list, user teams need to write rights for “board_admin” and read rights for the forum.<br>– To create qualitative projects themselves, user teams need read rights for “cr_project”, write rights for “board_admin” and read rights for the referenced forum.</li>
</ul>

<h2 id="managing-forums" class="wysiwyg-text-align-justify"><strong>Managing Forums</strong></h2>
<p class="wysiwyg-text-align-justify">The starting point for forum management is the menu <strong>Website → Panel websites → {Selected language version} → Forums</strong>. In this menu, you can edit forums, start new topics, and administrate ongoing discussions.</p>
<p class="wysiwyg-text-align-justify"><strong>Forum overview</strong></p>
<p class="wysiwyg-text-align-justify">The forum overview contains all forums created for the specified language version including name, description, and number of topics.</p>
<p class="wysiwyg-text-align-justify">If you want to view forums of another language version in a given installation with several websites and/or language versions, simply change the settings in the upper section of the dialog. You can use the View function to change the number of table columns displayed as well as the number of rows per page. You can search the content of the overview table.</p>
<p class="wysiwyg-text-align-justify"><strong>Editing options</strong></p>
<p class="wysiwyg-text-align-justify">If you click on the <strong>Create Forum</strong> button, the dialog for creating a new forum is opened.</p>
<p class="wysiwyg-text-align-justify">By clicking on the title of a forum, you open its list of topics. Click the title of the topic, and you will get to the individual posts.</p>
<p class="wysiwyg-text-align-justify">Use the icons in the “Actions” column to edit individual forums:</p>
<ul class="wysiwyg-text-align-justify">
<li>
<strong>Edit</strong>: Opens the dialog for changing forum properties. For detailed information on the properties.</li>
<li>
<strong>Edit topics</strong>: Opens the list of topics of the selected forum.</li>
<li>
<strong>Copy</strong>: Copies the selected forum. The posts are not copied, as this is usually not advisable.</li>
<li>
<strong>Export posts</strong>: Triggers an export of all posts of the forum.</li>
<li>
<strong>Assign bonus points</strong>: The form for assigning bonus points to the participants is open.</li>
<li>
<strong>Set rights</strong>: You can restrict the access rights of user teams to individual forums.</li>
<li>
<p><strong>Delete</strong>: The forum is deleted after a confirmation prompt. Please note that all data are irrevocably lost in the process. If you no longer need a certain forum, we recommend exporting the posts before deleting them.</p>
</li>
</ul>
